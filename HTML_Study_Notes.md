## 1. 웹 & HTML 기초

### 1.1 웹 동작 원리

| 단계 | 설명 |
|------|------|
| 요청 (Request) | 브라우저가 서버에 웹 페이지를 요청 |
| 처리 | 서버가 요청을 처리하고 웹 페이지를 준비 |
| 응답 (Response) | 서버가 파일들(HTML, CSS, JS 등)을 브라우저에 전송 |
| 렌더링 (Rendering) | 브라우저가 받은 파일들을 해석해서 웹 페이지로 변환 |

### 1.2 HTML이란?

- **HTML(HyperText Markup Language)**: 웹페이지의 구조와 내용을 정의하는 마크업 언어
- 태그 기반: `<태그명> </태그명>` 구조
- 구조화된 문서, 하이퍼링크, 플랫폼 독립적

### 1.3 HTML 문서 기본 구조

```html
<!DOCTYPE html>       <!-- HTML5 문서 선언 -->
<html>
  <head>              <!-- 사용자에게 안 보이는 메타 정보 -->
    <title>탭 제목</title>
  </head>
  <body>              <!-- 사용자가 실제로 보는 내용 -->
  </body>
</html>
```

### 1.4 태그 기본 구조

```
<태그명 속성="값"> 내용 </태그명>
```

- **여는 태그**: `<태그명>`
- **닫는 태그**: `</태그명>`
- **속성**: 태그에 추가 정보 제공
- **내용**: 태그 사이에 들어가는 실제 콘텐츠

### 1.5 개발자 도구

| 단축키 | 방법 |
|--------|------|
| Windows / Linux | `F12` |
| Mac | `Cmd + Option + I` |
| 공통 | 마우스 우클릭 → "검사" 선택 |

- **할 수 있는 것들**: HTML 구조 확인, CSS 스타일 분석, 실시간 수정, 오류 찾기, 성능 분석

## 2. 기본 태그

### 2.1 텍스트 관련

| 태그 | 설명 |
|------|------|
| `<h1>` ~ `<h6>` | 제목 (숫자 작을수록 크게 표시) |
| `<p>` | 문단 (paragraph) |
| `<br>` | 줄 바꿈 (닫는 태그 없음) |
| `<strong>` | 굵은 글씨 (중요한 내용) |
| `<em>` | 기울임 글씨 (강조하는 내용) |

### 2.2 목록

```html
<ul>         <!-- 순서 없는 목록 (불릿) -->
  <li>항목</li>
</ul>

<ol>         <!-- 순서 있는 목록 (번호) -->
  <li>항목</li>
</ol>
```

> `<li>`는 반드시 `<ul>` 또는 `<ol>` 안에서만 사용

### 2.3 링크 `<a>`

```html
<a href="연결할 주소">링크 텍스트</a>
```

| href 유형 | 예시 | 설명 |
|-----------|------|------|
| 외부 링크 | `https://example.com` | https:// 로 시작하는 완전한 URL |
| 내부 링크 | `page.html` | 같은 사이트 내 다른 페이지 |
| 앵커 링크 | `#목표` | 같은 페이지 내 특정 위치로 이동 |

- 앵커 링크 사용법:
```html
<a href="#section1">이동하기</a>
...
<h2 id="section1">목적지</h2>
```

### 2.4 이미지 `<img>`

```html
<img src="이미지 주소" alt="이미지 설명" width="300" height="200">
```

| 속성 | 설명 |
|------|------|
| `src` | 이미지 경로 (필수) |
| `alt` | 이미지를 볼 수 없을 때 표시되는 대체 텍스트 |
| `width` | 가로 크기 (픽셀) |
| `height` | 세로 크기 (픽셀) |

### 2.5 테이블

```html
<table>
  <thead>
    <tr>
      <th>제목 셀</th>   <!-- 굵게 + 가운데 정렬 -->
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>데이터 셀</td>
    </tr>
  </tbody>
</table>
```

| 속성 | 설명 |
|------|------|
| `colspan="값"` | 열 병합 |
| `rowspan="값"` | 행 병합 |

### 2.6 인용문 & 코드

| 태그 | 설명 |
|------|------|
| `<blockquote>` | 긴 인용문 (블록) |
| `<q>` | 짧은 인용문 (인라인) |
| `<cite>` | 인용 출처 표시 |
| `<pre>` | 긴 코드 블록 (원본 형태 그대로, 들여쓰기·줄바꿈 유지) |
| `<code>` | 짧은 코드 (인라인) |

### 2.7 구분선 & 레이아웃

| 태그 | 설명 |
|------|------|
| `<hr>` | 수평선 |
| `<div>` | 블록 레벨 컨테이너 (큰 영역 나누기) |
| `<span>` | 인라인 컨테이너 (문장 속 일부만 스타일 지정) |

## 3. 폼 & 입력 요소

### 3.1 `<form>`
- 웹페이지에서 사용자가 정보를 입력하고 서버로 전송할 수 있게 해주는 기능

```html
<form action="/submit" method="post">
  ...
</form>
```

| 속성 | 설명 |
|------|------|
| `action="주소"` | 폼 데이터를 전송할 서버 주소 |
| `method="get"` | URL 뒤에 데이터를 붙여서 전송 (보안 취약) |
| `method="post"` | 서버에 직접 전송 (권장) |

### 3.2 `<input>`
- 사용자 입력을 받는 필드

#### 3.2.1  type 종류

| type | 설명 |
|------|------|
| `text` | 일반 텍스트 |
| `password` | 비밀번호 (입력값을 ●로 표시) |
| `email` | 이메일 형식 |
| `number` | 숫자 |
| `checkbox` | 체크박스 (다중 선택, 각각 다른 name) |
| `radio` | 라디오 버튼 (단일 선택, 같은 name으로 그룹핑) |
| `hidden` | 화면에 안 보이지만 서버로 함께 전송 |

#### 3.3.2 주요 속성

| 속성 | 설명 |
|------|------|
| `name="값"` | 서버로 전송할 때 사용할 키 이름 |
| `value="값"` | 기본값 + 서버로 보내는 값 |
| `placeholder="문구"` | 입력창 안 회색 안내 문구 |
| `required` | 필수 입력 지정 |
| `readonly` | 읽기 전용 (수정 불가) |
| `maxlength="값"` | 최대 글자 수 |
| `minlength="값"` | 최소 글자 수 |
| `size="값"` | 입력창 크기 |
| `disabled` | 비활성화 |

### 3.4 드롭다운 `<select>`

```html
<select name="city" multiple size="3">
  <optgroup label="경상도">
    <option value="daegu" selected>대구</option>
    <option value="busan">부산</option>
  </optgroup>
</select>
```

| 태그 / 속성 | 설명 |
|-------------|------|
| `<select>` | 드롭다운 전체 컨테이너 |
| `<optgroup label="이름">` | 옵션 그룹화 |
| `<option value="값">` | 서버로 전송될 값 |
| `multiple` | 다중 선택 가능 (Windows: Ctrl + 클릭 / Mac: Command + 클릭) |
| `size="값"` | 화면에 보이는 항목 수 |
| `selected` | 기본 선택 항목 |

### 3.5 기타 입력 요소

- **`<textarea>`** — 여러 줄 텍스트 입력
```html
<textarea rows="5" cols="40" placeholder="내용을 입력하세요"></textarea>
```

- **`<button>`**
```html
<button type="submit">제출</button>   <!-- 폼 제출 (기본값) -->
<button type="reset">초기화</button>  <!-- 폼 초기화 -->
<button type="button">일반</button>   <!-- 일반 버튼 -->
```

- **`<label>`** — 입력 요소와 연결해 접근성·편의성 향상
```html
<!-- 방법 1: for와 id 연결 -->
<label for="username">아이디</label>
<input type="text" id="username">

<!-- 방법 2: label로 input 감싸기 -->
<label>아이디 <input type="text"></label>
```

## 4. 시맨틱 태그

### 4.1 시맨틱이란?

태그가 콘텐츠의 **의미**를 명확히 표현하는 것

**장점**
- 검색 엔진 최적화 (SEO)
- 웹 접근성 향상 (스크린 리더 등 보조 기술)
- 코드 가독성

### 4.2 페이지 구조

```
<body>
├── <header>   — 상단 영역 (로고, 사이트 제목)
├── <nav>      — 주요 내비게이션 링크
├── <main>     — 핵심 콘텐츠 (페이지당 1개)
│   ├── <section>   — 주제별 그룹 (각자 h태그 제목 필요)
│   │   └── <article>  — 독립 콘텐츠
│   └── <aside>     — 보조 콘텐츠
└── <footer>   — 하단 영역 (저작권, 연락처)
```

### 4.3 태그별 특징

| 태그 | 설명 | 주의사항 |
|------|------|----------|
| `<header>` | 문서/섹션의 상단 영역 | 중첩 불가 |
| `<nav>` | 주요 내비게이션 링크 모음 | |
| `<main>` | 페이지의 핵심 콘텐츠 | 페이지당 1개만 사용 |
| `<section>` | 주제별 콘텐츠 그룹 | 반드시 제목(h1~h6) 포함 |
| `<article>` | 독립적으로 배포 가능한 완전한 콘텐츠 | 다른 곳에 옮겨도 이해 가능해야 함 |
| `<aside>` | 보조 콘텐츠 | 없어도 메인 이해에 지장 없음 |
| `<footer>` | 문서/섹션의 하단 영역 | 여러 개 사용 가능 |

> **`<div>` vs 시맨틱 태그**: `<div>`는 의미 없는 컨테이너, 시맨틱 태그는 콘텐츠의 역할을 명시. 가능하면 시맨틱 태그 우선 사용!
