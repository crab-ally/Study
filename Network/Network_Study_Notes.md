# 네트워크

## 네트워크 기초

### 1. IP 주소 (IP Address)
> **비유:** 네트워크 세상의 '행정구역 주소' (이사를 가면 주소가 바뀌듯, 네트워크 환경이 바뀌면 IP도 변경됨)

인터넷이나 네트워크에서 장치들을 서로 구분하기 위해 부여하는 **논리적인 주소**

#### 주소 체계의 형태
* **IPv4 ($192.168.0.1$):** 숫자로 된 32비트 주소체계. 현재 가장 널리 쓰이지만, 전 세계 기기가 늘어나며 주소가 고갈되는 문제가 발생
* **IPv6 ($2001:0db8:85a3:0000:0000:8a2e:0370:7334$):** 주소 고갈 문제를 해결하기 위해 만든 128비트의 차세대 주소 체계

#### 공인 IP vs 사설 IP

* **공인 IP (Public IP):** ISP(인터넷 서비스 제공자)가 할당하며, 전 세계에서 유일한 '진짜 주소'.
  * 인터넷이라는 큰 도로에 직접 연결된 주소이므로, 외부 사용자가 우리 서버에 접속하려면 반드시 이 공인 IP를 찾아와야 한다.
* **사설 IP (Private IP):** 가정이나 회사 내부 네트워크(공유기 안쪽)에서만 사용하는 '우리끼리의 주소' ($192.168.x.x$, $10.x.x.x$ 등).
  * 건물 내부의 '101호, 102호' 같은 개념이라 외부에서는 이 주소로 직접 찾아올 수 없다. 상황에 따라 동적으로 변하기도 한다.
* **NAT (Network Address Translation):** 사설 IP를 가진 기기가 인터넷으로 나가려면, 공유기(라우터)가 사설 IP를 대표 공인 IP로 변환해 주는 기술인 **NAT**를 반드시 거쳐야 한다. 서버는 결국 요청을 보낸 이 공인 IP를 바라보게 된다.

#### 내 주소 확인법
* **공인 IP:** 네이버나 구글에 "내 IP 주소 검색"
* **사설 IP:** 터미널/CMD 창에서 `ipconfig` (Mac/Linux는 `ifconfig` 또는 `ip a`) 입력

### 2. MAC 주소 (Media Access Control Address)
> **비유:** 기기 고유의 '주민등록번호' (전 세계에서 유일하며 변경 불가능)

네트워크 카드(NIC, 랜카드)가 공장에서 만들어질 때 하드웨어에 고정되어 나오는 **물리적 주소** ($00:1A:2B:3C:4D:5E$ 형태)

#### IP 주소와 MAC 주소의 차이점

**인터넷 바다에서 목적지 네트워크를 찾아갈 때는 IP(3계층)**를 보고 가고, **같은 네트워크(LAN) 안에서 옆 기기로 최종 배달할 때는 MAC 주소(2계층)**를 쓴다.

| 구분 | MAC 주소 (물리 주소) | IP 주소 (논리 주소) |
| :--- | :--- | :--- |
| **목적** | 하드웨어(물리적) 식별 | 네트워크(논리적) 위치 식별 |
| **변경 가능성** | 변경 불가 (하드웨어 고정) | 상황에 따라 변경 가능 (동적/수동 할당) |
| **할당 방식** | 제조사에서 생산 시 미리 지정 | 네트워크 관리자가 지정 또는 DHCP가 자동 할당 |
| **OSI 계층** | **OSI 2계층** (데이터 링크 계층) | **OSI 3계층** (네트워크 계층) |

### 3. 랜선 (LAN 케이블)
> **비유:** 데이터가 실제로 이동하는 '물리적 도로'

아무리 소프트웨어 세팅이 잘 되어 있어도 현장에서 선이 부실하거나 규격이 안 맞으면 대형 장애가 발생한다.

#### 규격별 종류 (Category)
실무에서는 통신 속도, 최대 전송 거리, 그리고 구축 비용을 고려해 케이블을 선택

* **Cat.5e ($1\text{Gbps}$ / $100\text{m}$):** 가성비가 좋아 일반 가정이나 기가 인터넷을 쓰는 일반 사무실에서 여전히 가장 흔하게 쓰인다.
* **Cat.6 ($1\text{Gbps} \sim 10\text{Gbps}$ / $100\text{m}$):** 현재 신축 건물이나 사무실 인프라를 구축할 때 가장 권장하는 표준 규격 (단, $10\text{Gbps}$ 속도는 거리 $55\text{m}$ 이하에서만 보장됨)
* **Cat.6a ($10\text{Gbps}$ / $100\text{m}$):** 기업용 백본 네트워크나 데이터 센터 인프라에 필수적
* **Cat.7 / Cat.8 ($10\text{Gbps} \sim 40\text{Gbps}$):** 차폐가 극대화된 케이블로, 노이즈가 심한 공장이나 특수 서버실 환경에서 주로 쓰인다.

#### 차폐 방식 (노이즈 방어)
선 주변에 고전압 전선이 지나가면 전자기 간섭(노이즈)으로 데이터가 깨지기 때문에 상황에 맞는 차폐 방식 선택이 중요

* **UTP (Unshielded):** 보호막 없이 피복 안에 선만 꼬여있는 형태. 일반 가정/사무실은 이걸로 충분
* **FTP (Foiled):** 전체 케이블을 얇은 알루미늄 호일로 한 번 감싸 잡음을 방지
* **STP (Shielded):** 페어(Pair)별로 편조 쉴드 처리가 되어 있어, 전자파 간섭이 매우 심한 공장이나 대형 서버실 벽면 내부 인프라에 필수적

### 흐름 (컴퓨터로 구글에 접속할 때)

1. 내 컴퓨터가 구글 서버의 **공인 IP**($142.250.x.x$)로 가고 싶다는 데이터 패킷을 만듦.
2. 같은 방(LAN)에 있는 공유기에게 보내기 위해 공유기의 **MAC 주소**를 찾아 데이터 링크(2계층)로 전달함.
3. 패킷을 받은 공유기는 **NAT** 기술을 이용해 내 컴퓨터의 **사설 IP**를 **공인 IP**로 변환하여 외부 인터넷 세상으로 던짐.
4. 라우터들을 거쳐 구글의 **공인 IP**에 무사히 도착!
