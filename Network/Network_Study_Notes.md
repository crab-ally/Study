# Part 1. 기초 — 내 컴퓨터가 인터넷에 연결되는 원리

## 1. IP 주소 (IP Address) — 네트워크 세상의 행정구역 주소

> 💡 이사를 가면 우편 주소가 바뀌듯, 네트워크 환경(와이파이, 회사, 카페)이 바뀌면 IP도 바뀝니다.

IP 주소는 인터넷이나 네트워크에서 장치들을 서로 구분하기 위해 부여하는 **논리적인 주소**입니다. 물리적으로 정해진 게 아니라, 상황에 따라 "부여되는" 주소라는 점이 핵심입니다.

### 1-1. 주소 체계의 형태

| 구분 | 형태 | 특징 |
|---|---|---|
| **IPv4** | `192.168.0.1` | 32비트 숫자 주소. 현재 가장 널리 쓰이지만 전 세계 기기 수가 늘면서 **주소 고갈** 문제 발생 |
| **IPv6** | `2001:0db8:85a3:0000:0000:8a2e:0370:7334` | 128비트 차세대 주소 체계. IPv4 고갈 문제를 해결하기 위해 등장 |

> 🔧아직도 국내 대부분의 사내망은 IPv4 기반입니다. IPv6는 알아만 두시고, 실무에서는 IPv4 계산(서브넷 마스크 등)에 훨씬 더 많이 부딪히게 될 겁니다. 서브넷 마스크는 9장에서 자세히 다룹니다.

### 1-2. 공인 IP vs 사설 IP

이 둘을 구분 못 하면 나중에 포트포워딩, 방화벽 설정에서 반드시 막힙니다.

* **공인 IP (Public IP)**
  * ISP(인터넷 서비스 제공자)가 할당하는, 전 세계에서 유일한 "진짜 주소"
  * 외부에서 우리 서버로 접속하려면 반드시 이 주소를 알아야 함 (큰 도로에 바로 연결된 주소)

* **사설 IP (Private IP)**
  * 가정/회사 내부망(공유기 안쪽)에서만 쓰는 "우리끼리의 주소" (예: `192.168.x.x`, `10.x.x.x`)
  * 건물의 "101호, 102호" 같은 개념 — 외부에서 이 주소로 직접 찾아올 수 없음
  * 상황에 따라 동적으로 바뀔 수 있음

> 🔧 "내 컴퓨터 IP가 192.168.0.5인데 왜 외부에서 이 주소로 접속이 안 되나요?" — 신입 때 정말 많이 받는 질문입니다. 답은 간단합니다. 그건 **사설 IP**라서 인터넷 바다에서는 아예 보이지 않기 때문입니다. 이걸 해결해주는 기술이 바로 7장에서 다룰 **NAT**입니다.

### 1-3. 내 주소 직접 확인해보기

* **공인 IP 확인:** 포털 사이트에 "내 IP 주소" 검색
* **사설 IP 확인:**
  * Windows: `ipconfig`
  * Mac/Linux: `ifconfig` 또는 `ip a`

## 2. MAC 주소 (Media Access Control Address) — 기기의 주민등록번호

> 💡 IP가 "행정구역 주소"라면, MAC은 사람으로 치면 "주민등록번호"입니다. 전 세계에 단 하나뿐이고, 원칙적으로 변경이 불가능합니다.

네트워크 카드(NIC, 랜카드)가 공장에서 생산될 때 하드웨어에 고정되어 부여되는 **물리적 주소**입니다. `00:1A:2B:3C:4D:5E`와 같은 형태를 가집니다.

### 2-1. IP 주소 vs MAC 주소, 언제 무엇을 쓰는가

> **먼 곳(인터넷 바다)을 찾아갈 때는 IP, 같은 방(LAN) 안에서 옆 기기로 최종 전달할 때는 MAC**

| 구분 | MAC 주소 (물리 주소) | IP 주소 (논리 주소) |
|---|---|---|
| 목적 | 하드웨어 식별 | 네트워크 위치 식별 |
| 변경 가능성 | 변경 불가 (하드웨어 고정) | 상황에 따라 변경 가능 |
| 할당 방식 | 제조사가 생산 시 지정 | 관리자 수동 지정 또는 DHCP 자동 할당 |
| OSI 계층 | 2계층 (데이터 링크 계층) | 3계층 (네트워크 계층) |

> 🔧 공유기의 "MAC 주소 필터링" 기능은 등록된 MAC 주소를 가진 기기만 와이파이에 접속하도록 허용하는 보안 기능입니다. IP는 계속 바뀔 수 있지만 MAC은 고정이기 때문에 가능한 방식입니다.

## 3. 랜선 (LAN 케이블) — 데이터가 실제로 달리는 물리적 도로

> 💡 아무리 소프트웨어 설정이 완벽해도, 현장에서 선 하나가 부실하면 그날 하루 종일 장애 대응만 하게 됩니다.

### 3-1. 규격별 종류 (Category)

| 규격 | 속도 | 최대 거리 | 주 사용처 |
|---|---|---|---|
| **Cat.5e** | 1Gbps | 100m | 일반 가정, 기가 인터넷 쓰는 일반 사무실 (가성비 좋음) |
| **Cat.6** | 1~10Gbps | 100m (10Gbps는 55m 이내에서만 보장) | 신축 건물·사무실 인프라의 표준 규격 |
| **Cat.6a** | 10Gbps | 100m | 기업용 백본 네트워크, 데이터 센터 |
| **Cat.7 / Cat.8** | 10~40Gbps | - | 노이즈가 심한 공장, 특수 서버실 |

### 3-2. 차폐 방식 — 왜 필요한가

* **UTP (Unshielded):** 보호막 없음. 일반 가정/사무실은 이걸로 충분
* **FTP (Foiled):** 케이블 전체를 알루미늄 호일로 한 번 감싸 잡음 방지
* **STP (Shielded):** 페어별로 편조 쉴드 처리. 전자파 간섭이 극심한 공장·대형 서버실에 필수

> 🔧 인터넷 속도가 갑자기 느려졌다면, 소프트웨어보다 먼저 케이블 상태를 확인하세요. 오래된 Cat.5 케이블이 눌리거나 꺾여 있는 경우가 생각보다 흔합니다.

## 4. 전체 흐름 정리 — 내 컴퓨터가 구글에 접속할 때 벌어지는 일

```
[내 PC] --(랜선/와이파이)--> [공유기] --(NAT)--> [인터넷] --> [구글 서버]
```

1. 내 컴퓨터가 구글 서버의 **공인 IP**(예: `142.250.x.x`)로 가고 싶다는 데이터 패킷을 만든다.
2. 같은 방(LAN)에 있는 공유기에게 전달하기 위해, 공유기의 **MAC 주소**를 찾아 2계층으로 넘긴다.
3. 패킷을 받은 공유기는 **NAT** 기술로 내 컴퓨터의 **사설 IP**를 공유기의 **공인 IP**로 변환해 인터넷 세상으로 던진다.
4. 여러 라우터를 거쳐 최종적으로 구글의 **공인 IP**에 도착한다.

> 📌 사실 위 그림에는 하나 빠진 게 있습니다. 우리는 브라우저에 `142.250.x.x`가 아니라 `google.com`을 입력합니다. 이 도메인을 IP로 바꿔주는 과정이 바로 Part 2에서 다룰 **DNS**입니다.

## 5. TCP/IP 프로토콜 — 통신의 규칙과 약속

**프로토콜(Protocol)**이란 컴퓨터나 장치들이 서로 데이터를 주고받는 규칙과 약속입니다.

### 5-1. 대표적인 프로토콜

| 프로토콜 | 설명 |
|---|---|
| **HTTP** | 웹사이트를 볼 때 사용하는 프로토콜 |
| **TCP/IP** | 인터넷에서 데이터를 주고받는 기본 프로토콜. IP는 주소를 찾고, TCP는 데이터가 정확히 전달되도록 보장 |
| **FTP** | 파일을 주고받을 때 사용하는 프로토콜 |

### 5-2. TCP vs UDP — 전화 통화 vs 문자 메시지

> **TCP는 전화 통화**입니다. 상대방이 잘 받았는지 계속 확인하면서 대화합니다.  
> **UDP는 문자 메시지**입니다. 일단 보내고, 받았는지 신경 쓰지 않고 계속 보냅니다.

| 구분 | TCP | UDP |
|---|---|---|
| 연결 여부 | 연결형 (연결 후 전송) | 비연결형 (바로 전송) |
| 신뢰성 | 높음 (손실 시 재전송) | 낮음 (손실 시 무시) |
| 전달 순서 | 순서 보장 | 순서 보장 안 함 |
| 속도 | 느림 (확인 절차 필요) | 빠름 (확인 없이 전송) |
| 데이터 무결성 | 높음 | 보장 안 함 |
| 예시 | 웹사이트, 이메일, 파일 전송, 채팅 | 스트리밍 영상/음성, 온라인 게임 |

### 5-3. TCP는 누락된 패킷을 어떻게 알아채고 복구할까?

1. **누락 패킷 감지** — 모든 패킷에 순서 번호(sequence number)를 붙여, 수신 측이 빠진 번호를 즉시 파악
2. **재전송 요청 (ACK 기반)** — 연속해서 잘 받은 마지막 패킷까지만 ACK를 보내고, 발신 측은 일정 시간 ACK가 없으면 손실로 간주
3. **손실 패킷 재전송** — 발신 측이 자동 재전송하면, 수신 측은 순서대로 재조합해 응용 프로그램에 전달

> 🔧 화상회의 중 화면이 살짝 깨져도 통화는 안 끊기는 건 UDP가 "완벽함보다 실시간성"을 우선했기 때문입니다. 파일 다운로드는 한 바이트도 손실되면 안 되니 TCP를 씁니다.

## 6. 포트(Port)와 쿠키(Cookie)

### 6-1. 포트 — 같은 주소, 다른 서비스 구분하기

**포트**는 네트워크 통신에서 특정 서비스를 구분하기 위한 번호입니다.

* 예: `192.168.1.100:80` → `192.168.1.100`은 IP 주소, `80`은 포트 번호

> 💡 IP 주소가 "아파트 동 주소"라면, 포트는 "몇 호"에 해당합니다. 같은 건물(IP)이라도 101호(웹 서비스, 포트 80)와 102호(메일 서비스, 포트 25)는 완전히 다른 서비스입니다.

> 🔧 방화벽 설정에서 "포트를 연다/닫는다"는 표현을 정말 많이 씁니다. 특정 포트를 열어야 그 포트를 쓰는 서비스가 외부와 통신할 수 있습니다.

### 6-2. 쿠키 — 웹사이트가 나를 기억하는 방법

**쿠키**는 사용자가 웹사이트를 방문했을 때 브라우저에 저장되는 작고 단순한 텍스트 파일입니다.

**쿠키의 역할**
1. 로그인 상태 유지
2. 개인화된 사용자 경험 제공 (언어, 테마, 지역 설정)
3. 쇼핑몰 장바구니 기억
4. 웹사이트 이용 통계 수집

**작동 순서**
1. 사용자가 웹사이트 방문
2. 서버가 쿠키 생성 요청
3. 재방문 시 브라우저가 쿠키 전달
4. 서버가 쿠키 정보로 개인화된 콘텐츠 제공

> 🔧 "로그인했는데 자꾸 풀려요"라는 문의는 대부분 쿠키 저장이 차단되어 있거나(시크릿 모드), 쿠키 만료 시간이 짧게 설정된 경우입니다.

# Part 2. 심화 — 인터넷과 라우팅

## 7. DNS (Domain Name System) — 도메인을 IP로 바꾸는 통역사

> 💡 DNS는 전화번호부입니다. 우리는 사람 이름(도메인)만 알면 되고, 실제 전화번호(IP)는 전화번호부가 찾아줍니다.

사람이 이해하기 쉬운 도메인 주소를 컴퓨터가 이해하는 IP 주소로 변환해주는 시스템입니다.

```
ychoi.kr (도메인 이름) → DNS → 185.199.110.153 (IP 주소)
```

> DNS가 없다면, 웹사이트에 접속할 때마다 모든 IP 주소를 직접 기억하고 입력해야 합니다.

### 7-1. 동작 순서

```
브라우저에 주소 입력 → DNS 서버에 IP 주소 요청 → DNS 서버가 IP 주소 응답 → 브라우저가 해당 IP로 접속
```

### 7-2. DNS의 기타 역할

* 도메인 관리 및 하위 도메인(subdomain) 설정
* 메일 서버의 위치 지정 (MX 레코드)
* 서비스 위치 지정 (SRV 레코드)
* 보안 설정 (DNSSEC)

> 🔧 "사이트가 갑자기 안 열려요"라는 장애의 상당수는 서버가 죽은 게 아니라 **DNS가 응답을 못 하는 경우**입니다. 이럴 때 `nslookup 도메인명`이나 `dig 도메인명` 명령으로 DNS가 IP를 제대로 돌려주는지부터 확인하는 게 첫 번째 점검 습관입니다.

## 8. NAT (Network Address Translation) 심화 — 공인 IP를 아껴 쓰는 기술

앞서 4장에서 NAT를 잠깐 언급했죠. 여기서는 좀 더 실무적으로 들어가 보겠습니다.

내부 네트워크의 여러 장비가 **하나의 공인 IP 주소를 공유**해서 인터넷을 사용할 수 있게 해주는 기술입니다.

```
| 내부 장치들 (사설 IP) | ⇄ NAT 장비(공유기) ⇄ | 인터넷 (공인 IP) |
```

### 8-1. NAT를 쓰는 이유

1. **공인 IP 주소 부족 해결** — IPv4 주소는 한정되어 있는데, 집마다 회사마다 기기가 수십~수백 대이므로 전부 공인 IP를 줄 수 없음
2. **보안성 강화** — 외부에서는 내부 장치가 직접 보이지 않으므로, 일종의 방화벽 역할을 함께 수행
3. **내부 네트워크 관리 용이** — 내부 IP 체계를 회사 사정에 맞게 자유롭게 구성 가능

### 8-2. NAT의 종류

| 종류 | 방식 | 주 사용처 |
|---|---|---|
| **Static NAT (정적 NAT)** | 내부 IP ↔ 공인 IP를 1:1로 고정 매핑 | 외부에 항상 노출돼야 하는 서버 |
| **Dynamic NAT (동적 NAT)** | 다수의 내부 IP를 다수의 공인 IP **풀(pool)**과 매핑 | 공인 IP를 여러 개 보유한 기업망 |
| **PAT (Port Address Translation)** | 하나의 공인 IP를 다수의 내부 IP가 **포트 번호**로 구분해 공유 | 가정용 공유기 (가장 흔한 방식) |

> 🔧 우리 집 공유기에 컴퓨터, 스마트폰, TV가 동시에 인터넷을 쓸 수 있는 이유가 바로 **PAT** 덕분입니다. 공유기는 각 기기의 요청을 서로 다른 포트 번호로 구분해서 관리하기 때문에, 공인 IP는 하나뿐이어도 문제없이 동작합니다.

## 9. 서브넷 마스크 (Subnet Mask) — 네트워크를 쪼개는 기준

> 💡 큰 아파트 단지(전체 네트워크)를 몇 동, 몇 호(서브넷)로 나누는 것과 같습니다.

IP 주소에서 **네트워크 영역**과 **호스트 영역**을 나누는 기준입니다.

```
IP 주소 = [네트워크 영역 | 호스트 영역]
```

### 9-1. 예시로 이해하기

`192.168.10.20`이라는 IP에 서브넷 마스크 `255.255.255.0`이 적용되어 있다면?

* 처음 세 부분(`192.168.10`) → **네트워크 영역**
* 마지막 부분(`20`) → **호스트 영역**

→ 이 IP는 "192.168.10 네트워크"에 속한 "20번 호스트"가 됩니다.

### 9-2. 서브넷을 나누는 이유

1. **네트워크 분할** — 하나의 큰 네트워크를 여러 작은 네트워크(서브넷)로 나눠 관리하기 쉽게 함
2. **IP 주소 효율적 사용** — 필요한 만큼만 나눠 써서 낭비를 줄임
3. **트래픽 제어 및 보안 향상** — 서브넷별로 관리가 용이해져 성능과 보안 개선

### 9-3. 자주 쓰는 서브넷 마스크 / CIDR 표기

| 서브넷 마스크 | CIDR 표기법 | 설명 |
|---|---|---|
| 255.0.0.0 | /8 | 네트워크 8비트, 호스트 24비트 |
| 255.255.0.0 | /16 | 네트워크 16비트, 호스트 16비트 |
| 255.255.255.0 | /24 | 네트워크 24비트, 호스트 8비트 (**가장 흔함**) |
| 255.255.255.128 | /25 | 네트워크 25비트, 호스트 7비트 |

> 🔧 회사에서 "개발팀은 192.168.1.0/24, 인사팀은 192.168.2.0/24를 쓴다"처럼 부서별로 대역을 나누는 이유가 바로 이것입니다. 서브넷이 다르면 기본적으로 서로 다른 네트워크로 취급되어, 트러블슈팅과 보안 정책 적용이 훨씬 쉬워집니다.

## 10. QoS (Quality of Service) — 트래픽에도 우선순위가 있다

네트워크 트래픽 중 특정 데이터의 속도나 품질을 보장하거나 우선순위를 정하는 기술입니다. 중요한 데이터는 빠르고 안정적으로, 덜 중요한 데이터는 나중에 처리합니다.

### 10-1. QoS의 주요 기능

1. **대역폭(Bandwidth) 관리** — 중요한 서비스에 더 많은 대역폭 배정
2. **우선순위(Priority) 관리** — 중요한 트래픽이 먼저 처리되도록 함
3. **지연(Latency) 및 지터(Jitter) 관리** — 지연이나 불규칙한 전송 속도 변화 최소화
4. **패킷 손실(Packet loss) 관리** — 중요한 데이터 패킷의 손실 방지

### 10-2. 주의사항

* 모든 공유기/네트워크 장비가 QoS를 지원하는 것은 아님
* 공유기 설정 페이지에서 QoS 기능을 별도로 활성화해야 함
* 우선순위를 잘못 설정하면 오히려 네트워크 성능이 떨어질 수 있음

> 🔧 재택근무 중 화상회의는 끊기는데 가족이 다른 방에서 대용량 파일을 다운로드하고 있다면, 공유기 QoS 설정에서 화상회의 트래픽에 우선순위를 부여하는 것으로 상당 부분 해결됩니다.

## 11. 망중립성 (Net Neutrality) — 인터넷을 다루는 원칙

인터넷에서 주고받는 모든 데이터(트래픽)를 통신사가 **차별 없이 공평하고 동등하게** 처리해야 한다는 원칙입니다.

### 11-1. 핵심 원칙

* **차별금지** — 특정 사이트나 서비스의 트래픽을 제한하거나 차단하면 안 됨
* **동등 접근** — 모든 콘텐츠와 서비스에 동등하게 접근할 수 있어야 함
* **공정성** — 인터넷 서비스 제공자는 모든 사용자와 서비스에 대해 중립적이어야 함

> 📌 QoS는 "내 네트워크 안에서" 우선순위를 정하는 기술이지만, 망중립성은 "통신사가 인터넷 전체 트래픽을 다룰 때" 지켜야 할 정책적 원칙이라는 차이를 기억해두면 좋습니다.
