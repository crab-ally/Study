# Stable-Baselines3

강화학습 알고리즘을 직접 수식부터 구현하지 않고도, 검증된 형태로 바로 사용할 수 있게 해주는 라이브러리

| | 설명 |
|---|---|
| **Pytorch** | 직접 강화학습 알고리즘을 만드는 도구 |
| **Stable-Baselines3** | Pytorch를 기반으로 강화학습 알고리즘을 구현해 놓은 라이브러리 |

---

**핵심개념**

| 요소 | 설명 |
| --- | --- |
| Environment | Agent가 상호작용하는 세계 |
| Observation | Agent가 보는 입력값 |
| Action | Agent가 내리는 결정 |
| Reward | 행동에 대한 즉각적인 피드백(점수) |
| Episode | 한 번의 시도 |

**전체흐름**

```
1. 환경을 만든다.
2. 환경을 `reset()` 해서 초기 관측을 받는다.
3. 정책이 관측을 보고 행동을 고른다.
4. 환경이 행동을 받아 다음 상태와 보상을 돌려준다.
5. 이 데이터를 모아서 알고리즘이 정책을 업데이트한다.
6. 반복한다.
```

---

**알고리즘**

| 알고리즘 | 설명 |
|---|---|
| DQN | 이산 행동 공간용 알고리즘 |
| A2C | replay buffer 없이 학습하는 정책 기반 계열 알고리즘 |
| PPO | 업데이트가 너무 크게 변하지 않도록 clipping을 사용하는 알고리즘 |

> 보상해킹: AI가 진짜 목표를 달성한 게 아니라, 보상 함수의 허점을 이용해서 보상만 높이는 현상

---

**종료**

- `terminated`: 목표 달성, 실패 등 **문제 자체가 끝난 상태**
- `truncated`: 시간 제한 등 **외부 조건으로 잘린 상태**
