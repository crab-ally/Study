# Linear Algebra

## 1. 기본

로봇은 세상을 좌표와 방향으로 인식한다. 이를 수학적으로 표현하고 계산하는 핵심 도구가 선형대수이다.

### 1-1. 점 vs 벡터

- 점(Point): 위치
- 벡터(Vector): 이동량, 변화량 (크기 + 방향)

### 1-2. 기본 용어

- frame: 기준 좌표계
- transform: 한 좌표계의 점을 다른 좌표계로 변환
- unit vector: 크기가 1인 벡터 (방향만 유지)

### 1-3. 기본 연산

- 덧셈: `(1, 2) + (3, 1) = (4, 3)` → 로봇 이동
- 뺄셈: `(4, 3) - (1, 2) = (3, 1)` → 목표까지 방향 계산
- 크기: `sqrt(x^2 + y^2)` → 목표까지 거리 계산
- 스칼라배: `2 * (1, 2) = (2, 4)` → velocity scaling

> velocity scaling: 이동 = 방향 * 속도

## 2. 내적

두 벡터가 얼마나 같은 방향을 향하고 있는지
→ 로봇이 목표를 보고 있는지

`A·B = |A||B|cosθ`

- 양수: 같은 방향
- 0: 수직
- 음수: 반대 방향

## 3. 행렬

좌표를 변환하는 도구

```python
# 행렬 * 벡터
A= [[a,b],
    [c,d]]
v= (x,y)

결과= (ax+by, cx+dy)
```

### 3-1. 행렬 기본 연산

- Scaling(확대/축소) → 속도, 센서 거리 scaling

```python
# 확대/축소 행렬
M = [[2, 0],
     [0, 2]]
# 벡터 v=(3,2) 를 2배 확대
v_new = M * v  # (6,4)
```

- 방향 뒤집기

```python
# 방향 뒤집기 행렬
M = [[-1, 0],
     [ 0,1]]
# 벡터 v=(3,2) 를 방향 뒤집기
v_new = M * v  # (-3,2)
```

- 회전 → 로봇 기준 이동을 지도 기준 이동으로 변환

```python
# 회전 행렬
M = [[cosθ, -sinθ],
     [sinθ,  cosθ]]
# 벡터 v=(x,y) 를 θ 만큼 회전
v_new = M * v
```

## 4. 좌표계(Frame)

`map → odom → base_link → sensor`

| frame | 설명 |
|---|---|
| map | 지도 기준 (고정) |
| odom | 이동 누적 기준 |
| base_link | 로봇 중심 |
| sensor | 센서 기준 |

### 4-1. tf

ROS2의 좌표계(Frame) 관리 시스템.
필요한 좌표 변환을 자동으로 계산해줌.

```python
# 데이터 흐름
joint_state_publisher : 관절 각도 발행
        ↓
    /joint_states
        ↓
robot_state_publisher : URDF와 관절 상태를 이용해 각 링크의 위치 계산
        ↓
   /tf, /tf_static : 계산된 좌표계를 전체 시스템에 제공
```
